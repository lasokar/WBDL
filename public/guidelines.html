Guidelines

Submission Rules

  • Submissions must have audible clicks/taps or a handcam.
  • Videos may not be edited in any way if that affects important gameplay elements.
    This includes but is not limited to: cropping, cutting, audio modifications, and screen visibility.
  • Submissions have to be done on the browser version of GD, and cannot utilize any website/host that isn't on the list below:
    web-dashers.github.io webdashers.dev
  • You may only use the following mods in your video:
    Show Percentage
    Percentage Decimals
    Startpos Switcher
    Show FPS
    Show CPS
    Solid Wave Trail
    Hitboxes on Death
  • You may bulk submit (aka use the same video for multiple submissions), but providing timestamps is required.
  • Submit using this form.

Verification Rules

  • Levels will be created and published on the official GD servers.
  • Levels must be playable with only the supported objects on Web Dashers.
    Note: If the level doesn't exactly match the original on the servers due to it using unsupported objects (like missing a few blocks or decoration), that's fine as long as it remains beatable and is mostly functional.
  • Levels must be 30 seconds long minimum. End-screens or long auto parts do not count towards this time.
  • Aim for decent gameplay and visual quality. Doesn't have to be rate worthy, but no recent tab stuff. Put some effort into creating levels...
  • Buffed, nerfed, or modified versions of existing list levels will NOT be accepted.
  • Upon completing the level, submission rules also apply.

Impossible List Rules

  • Levels may be showcased on real GD rather than Web Dashers, as the botting tool available on the web version is still very unstable.
  • Levels must also be "humanly possible". This means no sections should require higher than 20cps.
  • Aim to be harder than the current top 1 WBDL. Levels that were placed on the ILL prior to a new top 1 being verified are allowed to remain.
  • The regular WBDL submission rules still apply.
